# Sheaf Knowledge Marketplace — 中远期战略调研

> Created: 2026-05-23 | Status: Draft v0.1

## 1. 产品愿景

**一句话**：让知识工作者通过 Sheaf 采集、整理的知识变成可确权、可交易、可组合的链上资产。

**核心命题**：知识经济时代，知识生产者的收益远低于平台。Sheaf 做的不只是「采集工具」，而是要让知识创作者对自己的劳动成果拥有真正的所有权和变现能力。

## 2. 竞品地图

### 2.1 Story Protocol（$140M 融资，a16z 领投）

| 维度 | 分析 |
|------|------|
| **定位** | IP（知识产权）管理的 Layer 1 区块链 |
| **核心能力** | IP 代币化 + 可编程许可 + 衍生品图谱自动版税分配 |
| **许可模型** | 4 级分层：开放使用 → 非商业再创作 → 商业使用 → 商业再创作 |
| **AI 关联** | 专门解决 AI 训练数据的合规授权问题 |
| **适用场景** | 创作者经济、AI 数据集、生物医药专利、品牌 IP |
| **与 Sheaf 的关系** | ⭐ **最强竞品也是最可能的合作方** — Story 做基础设施层，Sheaf 做应用层（知识包生产+消费） |

**关键洞察**：Story 不做具体应用，只做 IP 基础设施。这意味着 Sheaf 可以作为 Story 生态的「知识包生产工具」—— 用户用 Sheaf 采集+精炼知识，然后通过 Story 的许可框架上链确权。

### 2.2 Ocean Protocol（去中心化数据交易）

| 维度 | 分析 |
|------|------|
| **定位** | 数据资产化和交易协议 |
| **核心机制** | 数据代币化 + Compute-to-Data（数据不出域计算）+ OCEAN 代币激励 |
| **适用场景** | AI 数据集、科研数据、IoT 数据 |
| **与 Sheaf 的关系** | 互补 — Ocean 做数据交易，Sheaf 做知识（而非原始数据）交易。知识包 = 数据的提炼物 |

### 2.3 Lens Protocol / Farcaster（去中心化社交）

| 维度 | 分析 |
|------|------|
| **定位** | 去中心化社交图谱协议 |
| **核心机制** | 内容 NFT 化 + 关注关系链上化 + 算法可替换 |
| **适用场景** | 创作者经济、社交内容确权 |
| **与 Sheaf 的关系** | 轻度关联 — Lens 做的是社交内容，Sheaf 做的是深度知识。但知识市场的「发现」和「推荐」可以借鉴社交图谱 |

### 2.4 Arweave（永久存储）

| 维度 | 分析 |
|------|------|
| **定位** | 区块链永久存储网络 |
| **核心机制** | 一次付费永久存储 + Proof of Access 共识 |
| **与 Sheaf 的关系** | **底层存储方案** — 知识包的链上存储可以选 IPFS（临时）或 Arweave（永久）|

## 3. 技术方案设计

### 3.1 Knowledge Pack（知识包）标准

```yaml
# knowledge-pack.yaml v0.1
metadata:
  id: "kp_abc123"
  title: "AI Agent 系统设计方法论"
  author: "0x1234...abcd"         # 创作者钱包地址
  created: "2026-05-23"
  version: "1.0.0"
  license: "commercial-remix"     # Story Protocol 许可类型
  price: 5.00                     # USDC
  
content:
  cards: 42                        # 知识卡片数量
  sources:                         # 原始来源追溯
    - url: "https://..."
      title: "Codex-maxxing"
      accessed: "2026-05-23"
  tags: ["AI Agent", "自动化", "Codex"]
  language: "zh-CN"
  
provenance:
  ipfs_hash: "Qm..."              # 内容 IPFS 哈希
  story_ip_id: "0x..."            # Story Protocol IP ID
  signature: "0x..."              # 作者签名
  
quality:
  score: 4.2                      # 社区评分
  downloads: 0
  reviews: []
```

### 3.2 技术架构（4 层）

```
┌──────────────────────────────────────┐
│         应用层 (Sheaf App)            │
│  知识包浏览 / 搜索 / 交易 / 推荐系统    │
├──────────────────────────────────────┤
│         协议层 (Story / 自建)          │
│  IP 注册 / 许可管理 / 版税分配          │
├──────────────────────────────────────┤
│         存储层 (IPFS + Arweave)       │
│  知识包内容永久存储 / 哈希校验           │
├──────────────────────────────────────┤
│         结算层 (Ethereum / L2)        │
│  支付 / 质押 / 代币经济                 │
└──────────────────────────────────────┘
```

### 3.3 两种技术路径

| 路径 | 方案 A：Story 生态集成 | 方案 B：独立构建 |
|------|----------------------|----------------|
| **开发成本** | 低（复用 Story 基础设施） | 高（自建链/合约） |
| **自主性** | 中（依赖 Story 协议） | 高 |
| **上市速度** | 快（3-6 个月原型） | 慢（12+ 个月） |
| **用户门槛** | 中（需 Story 生态钱包） | 高（需自建用户群） |
| **推荐** | ✅ **推荐初期采用** | 适合规模化后迁移 |

**建议策略**：先做 Story 生态集成（MVP），验证需求后再考虑独立构建。

## 4. 商业模型

### 4.1 收入来源

| 来源 | 说明 | 预期占比 |
|------|------|---------|
| 交易手续费 | 知识包成交的 2-5% | 40% |
| 创作者订阅 | 高级工具/分析功能月费 | 30% |
| 企业 API | 团队知识管理 SaaS | 20% |
| 代币经济 | 质押/流动性挖矿 | 10% |

### 4.2 冷启动策略

1. **种子用户**：学术圈（知识密集 + 付费意愿强）
2. **首批知识包类型**：论文解读集、技术教程包、行业研报合集
3. **激励机制**：早期创作者获得平台代币空投
4. **合作伙伴**：Story Protocol 官方生态 + a16z portfolio 协同

## 5. 风险与挑战

### 5.1 法律风险（⚠️ 关键）

| 风险 | 影响 | 应对 |
|------|------|------|
| 中国对加密货币交易的监管 | 可能无法在国内直接运营代币经济 | 双轨制：国内用积分体系，海外用链上代币 |
| 知识产权归属争议 | 采集内容的版权不清晰 | 强调 Sheaf 是「加工工具」，版权归原始作者 |
| NFT/数字藏品监管 | 2026 年中国对 NFT 持审慎态度 | 走「数字版权登记」而非「NFT 交易」路线 |

### 5.2 技术风险

| 风险 | 应对 |
|------|------|
| IPFS 存储不稳定 | 重要内容 Arweave 备份 |
| Gas 费过高 | 使用 L2（Base/Optimism） |
| 知识包质量参差不齐 | 社区审核 + AI 质量评分 |

### 5.3 市场风险

| 风险 | 应对 |
|------|------|
| 用户对 Web3 门槛敏感 | 提供「无感知 Web3」体验（托管钱包） |
| Story Protocol 发展不如预期 | 保持协议层可替换 |
| 传统知识付费平台竞争（得到/知乎） | 差异化：AI 自动采集 + 链上确权 |

## 7. 结论与建议

### 7.1 战略定位

知识市场是 Sheaf 的**「终局愿景」**，但不应该是当前阶段的焦点。它的正确位置是：

```
Sheaf 产品三层蛋糕：
  🎂 第三层（v2.0+）：知识市场 — 去中心化知识交易 + Web3 所有权
  🍰 第二层（v1.0） ：知识网络 — 插件生态 + Obsidian 集成 + 知识图谱
  🧁 第一层（当前） ：知识引擎 — CLI 采集 + 结构化卡片 + 本地存储
```

### 7.2 近期行动建议

1. **不要现在做**。当前 Focus = Alpha 发布（PyPI + CLI 稳定）
2. **持续跟踪 Story Protocol**。它的生态发展直接决定我们的集成路径
3. **暗中积累**。在开发 Sheaf CLI 时，确保 Knowledge Pack 格式的可扩展性
4. **用户调研**。Beta 阶段验证「用户是否愿意为知识包付费」这个核心假设

### 7.3 差异化护城河

Sheaf 知识市场 vs 传统知识付费（得到/知乎/小报童）：

| 维度 | 传统平台 | Sheaf 知识市场 |
|------|---------|--------------|
| 内容生产 | 人工撰写 | **AI 辅助采集+精炼** |
| 所有权 | 平台控制 | **链上确权，用户所有** |
| 定价 | 平台抽成高 | **智能合约，低手续费** |
| 内容形式 | 长文/音频 | **结构化知识卡片包** |
| 推荐算法 | 黑盒 | **可替换/开源** |
| 衍生创作 | 无追踪 | **自动版税分配** |

**核心差异化**：Sheaf 不卖「内容」，卖的是「AI 精炼过的结构化知识」+ 「链上所有权证明」。这是传统平台做不了的。

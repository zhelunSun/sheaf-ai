## 1. 为什么是 Glean

### 词源与意象

**Glean** /ɡliːn/ — 源自古法语 *glener*，本意是"拾穗"：农人在收割后的麦田里，弯腰捡拾遗落的麦穗。

这个意象完美映射了产品的核心行为：

| 农人拾穗 | Glean 用户 |
|---------|-----------|
| 麦田已收割，但总有遗落的麦穗 | 互联网信息已读过，但总有值得留下的片段 |
| 弯腰，捡起，放进篮子 | 粘贴链接，一键收藏 |
| 一穗不多，但日积月累够一家人过冬 | 一篇不多，但积少成多构建完整知识库 |
| 不是大规模收割（那是农夫的工作） | 不是系统化学习（那是课程/书本的工作） |
| 是"捡"——低成本、低门槛、可持续 | 是"攒"——零摩擦、自动分类、Agent 帮你消化 |

### 为什么 Glean 比其他候选好

| 候选 | 问题 |
|------|------|
| Loom | 已被 Atlassian 收购（视频工具），品牌冲突 |
| Amber | 琥珀 → 封存感太强，暗示"冻结"而非"活用" |
| Sink | 下沉 → 消极意象，暗示信息"沉没" |

**Glean 的优势**：
1. **动词性** — "I glean" 是一个动作，匹配产品的核心交互（收藏=动作）
2. **农耕意象** — 自然、朴素、有温度，与 AI 产品的冰冷感形成对比
3. **"拾"的哲学** — 不贪多，不焦虑，每天捡几穗就好 → 天然反信息焦虑
4. **双关** — to glean 也意味着"慢慢收集信息/知识"，现代英语直接对应产品功能
5. **简短有力** — 5 个字母，单音节，好记好拼

## 2. 核心叙事：从"收藏即坟墓"到"拾穗即积累"

### 旧叙事（Universal Collector）

> 信息过载 → 收藏即坟墓 → Agent 替人消化 → 人只选信息源

功能导向，强调"Agent 替你干活"。

### 新叙事（Glean）

> 信息像收割后的麦田，到处都是遗落的麦穗。
> 你不需要收割整片麦田——只需要弯腰，捡起那一穗。
> 每天 5 分钟，捡几穗，扔进篮子。
> Agent 帮你脱粒、分类、归仓。
> 日积月累，你的粮仓满满当当。
> 这不是收藏——这是 **gleaning**。

**叙事升级点**：
- 从"解决焦虑" → "享受积累"
- 从"Agent 替你干活" → "你负责捡，Agent 负责加工"
- 从"工具" → "仪式感"

## 3. 游戏化系统设计（Glean-as-a-Game）

### 3.1 核心洞察

> "攒"是人性最深层的满足感来源之一。
> 存款账户的数字在涨、游戏角色的等级在升、集换式卡牌越集越全——
> 这些都是同一个心理学机制：**可积累的微小奖励**。

Glean 的游戏化不需要外挂——**知识积累本身就是天然的游戏循环**。

### 3.2 游戏循环设计

```
粘贴链接 → 自动分类（惊喜：AI 识别了什么主题？）
         → 摘要生成（满足：5秒看完核心）
         → 入库（积累：+1 知识卡片）
         → 标签关联（发现：这篇和上周那篇有关联！）
         → 里程碑触发（奖励：连续7天 / 收藏100篇 / 覆盖10个主题）
```

### 3.3 具体机制

#### 🌾 每日拾穗（Daily Glean）

| 机制 | 设计 |
|------|------|
| **每日签到** | 每天收藏至少 1 篇 = "今日拾穗完成" |
| **连续打卡** | 3天/7天/30天连续 → 不同等级徽章 |
| **每周拾穗报告** | Agent 自动生成："本周你捡了 12 穗，最热门主题是 AI Agent" |

#### 🧺 篮子（The Basket）

| 机制 | 设计 |
|------|------|
| **主题篮子** | 每个主题（如"AI Agent"/"遥感"/"投资"）是一个"篮子" |
| **篮子进度** | 每个篮子有填充度：5篇=萌芽 / 20篇=成长 / 50篇=丰盛 / 100篇=专家 |
| **篮子关联** | 当两个篮子有足够交叉标签 → "篮子串联"：发现跨领域洞察 |

#### 🏅 里程碑（Milestones）

| 里程碑 | 名称 | 触发条件 | 奖励 |
|--------|------|---------|------|
| First Sheaf | 第一穗 | 收藏第 1 篇 | 解锁个人知识库 |
| Week One | 一周拾穗人 | 连续 7 天收藏 | 自定义标签颜色 |
| The Hoarder / The Curator | 囤积者 / 策展人 | 50 篇 / 100 篇 | 高级统计面板 |
| Renaissance | 文艺复兴 | 覆盖 5+ 个不同主题 | 跨领域洞察报告 |
| Deep Diver | 深潜者 | 某主题 ≥ 20 篇 | 主题知识图谱 |
| The Gleaner | 拾穗人 | 连续 30 天 | 终极徽章 + 年度报告 |

#### 🌊 知识河流（Knowledge Stream）

| 机制 | 设计 |
|------|------|
| **时间线视图** | 你的收藏像一条河，每天几滴水，汇成江河 |
| **趋势浮现** | "最近 7 天你捡了很多 AI Agent 的穗——要不要看看这个主题的全景？" |
| **知识回流** | Agent 主动推送："你 3 个月前收藏的 X 话题，最近有新进展" |

### 3.4 游戏化原则

1. **零强制** — 没有惩罚机制，只有奖励。断签不扣分，只是重新计数
2. **天然驱动** — 游戏化建立在真实知识积累上，不是虚假成就感
3. **隐私优先** — 所有数据本地，游戏进度不与任何人比较（除非用户选择分享）
4. **Agent 是伙伴，不是裁判** — Agent 帮你发现成就，不评判你

## 4. 中文产品名候选

### 方案 A：拾穗（Shísuì）

- **直译 Glean**，保留农耕意象
- 文艺、有古意（让人想到米勒的《拾穗者》）
- 但：两字读音不够响亮，"穗"字日常使用少

### 方案 B：攒（Zǎn）

- **单字品牌**，极简，像"钉""飞书"
- 直接对应核心行为："攒知识"
- 口语化，亲切："今天你攒了吗？"
- 但：单字商标注册难度高

### 方案 C：拾光（Shíguāng）

- 谐音"时光"，暗含"拾取信息之光"
- 有"收集珍贵时刻"的温暖感
- 但：和"拾光照相馆"等品牌冲突风险

### 方案 D：攒穗（Zǎnsuì）

- "攒"+"穗"的组合
- 保留农耕意象 + 口语化
- 但：两个少见字组合，认知成本高

### 方案 E：谷仓（Gǔcāng）

- 从"拾穗"到"入仓"的完整隐喻
- 产品名=你的知识谷仓
- 但：更像存储产品（网盘感）

### 推荐

**短期**：英文用 **Glean**，中文暂时不换（保持 Universal Collector 或直接用 Glean）
**待确认**：Sir 从上面候选中选一个，或提出新想法

> 选项 A（拾穗）最忠于 Glean 原意，选项 B（攒）最口语化有传播力。
> 如果要走极简路线，甚至可以考虑"攒"作为中文独立品牌名——
> "今天攒了什么？" 天然就是一个好 slogan。

## 6. 潜在风险与缓解

| 风险 | 缓解 |
|------|------|
| Glean 名称已被占用（Glean.com 是企业搜索公司） | 我们是个人知识工具，品类不同；加后缀如 glean-pk / glean-collector |
| 游戏化显得不严肃 | 保持极简：不做花哨动画，只在 CLI/MCP 输出中轻触 |
| 农耕隐喻在不同文化中不 work | "Gleaning" 在英语里有通用含义（收集信息），不限于农业 |
| 过度设计，分散核心功能精力 | 游戏化是 Phase 3，当前只沉淀 idea，不实现 |

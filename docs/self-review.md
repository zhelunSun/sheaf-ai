### 1.2 ❌ 已知技术不足

#### P0：影响 MVP 核心体验的

| 不足 | 影响 | 建议修复方案 |
|------|------|-------------|
| **正文截断**：`text[:6000]` 硬截断，长文章丢失后段关键信息 | `doc/ch1-autonomous-plan.md` 类长文会丢失后半 | 改为动态截断：优先保留开头+结尾，或按段落分块递进 |
| **无去重**：同一 URL 可重复收录，产生多条重复条目 | 用户手滑或链接重发导致重复 | P1 加 URL hash 去重；P2 加内容相似度去重 |
| **API 失败无重试**：LLM 调用异常直接返回默认 fallback | 偶尔出现时产出质量极差 | 加 1-2 次自动重试（指数退避） |
| **批量处理缺失**：单次只能处理一个 URL | 用户想一次性发 10 篇文章需要手动重复 | 支持多 URL 参数 + 批量索引追加 |

#### P1：影响体验但可接受的

| 不足 | 影响 | 建议 |
|------|------|------|
| **纯关键词搜索**：不支持语义检索，\|清华大学\|搜不到\|清华\| | 查询体验受限 | Phase 1 上 Embedding + ChromaDB |
| **manual_text 模式无标题提取**：用户手动粘贴正文时 title 为空 | 手动模式标题缺失 | 从正文开头 200 字用 LLM 提取标题 |
| **索引无版本控制**：`index.jsonl` 只增不删，误操作无法回滚 | 无法删除错误条目 | 加 `status: deleted` 软删除逻辑 |
| **无定时任务集成**：收藏后需用户手动触发处理 | 自动化程度不够 | 对接 WorkBuddy automation |
| **并发线程安全**：同时收录多篇时 `index.jsonl` 写入冲突 | 边界情况 | 加文件锁或改用 SQLite |

### 2.3 可感知的不足

#### a) 语言选择不统一
当前输出全部英文，但用户是中文母语者。存在"摘要英文、标签中文、元数据结构英文"的混搭。建议：
- **一句话摘要** → 中英双语（中文为主，英文为辅）
- **结构化要点** → 中文
- **标签** → 中文关键词
- 在 schema 中增加 `language` 字段控制

#### b) "与你相关" 段逐渐失去新鲜感
当前 Prompt 固定写死了用户背景（"AI/remote sensing researcher, AI/Web3 investment"）。如果每篇文章都按这个模板生成，1-2 周后会出现同质化。建议：
- 从实际收藏历史中提取用户兴趣动态
- 在 Prompt 中注入近期高频标签

#### c) 子主题命名一致性
相同的子主题可能被不同 LLM 调用命名为不同名称。例如"多智能体系统" vs "多智能体规划" vs "Multi-Agent 系统"。建议：
- 对子主题做归一化（缓存已有子主题，新命名做相似度匹配）
- 或直接不限制，让用户后续通过查询来发现

#### d) 来源作者提取不稳定
LLM 从正文中提取作者名时，硅谷那篇提取了"孟醒 (Meng Xing), Partner at 5Y Capital"（优秀），但其他三篇提取为空（正常，微信文章作者信息不在正文中）。建议：
- 从微信公众号的 meta tag 提取 `wechat_author`
- 或从正文开头 200 字 + 末尾署名区用规则提取

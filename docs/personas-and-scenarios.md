# Universal Collector — 用户画像与使用场景

> 3 个典型 Persona，每个配具体的一天使用时间线
> 日期：2026-05-16

---

## Persona 1：林哲 — AI/遥感方向博士生

### 基本信息

| 维度 | 描述 |
|------|------|
| 年龄 | 25 |
| 身份 | 清华大学地学系博士生，AI+遥感交叉方向 |
| 城市 | 北京 |
| 技术水平 | 高。Python/PyTorch 全栈，会折腾 Agent 框架，日常使用 Claude/GPT/DeepSeek |
| 信息饮食 | 中英文混合，日均 50-100 条信息线索 |
| 工具栈 | 微信、浏览器、Gmail、Notion、Obsidian、VS Code、终端、WorkBuddy |

### 典型的一天

**07:30 — 通勤刷手机**
地铁上刷微信公众号。看到三篇想读的：
- 「DeepSeek 开源了新的 MoE 架构」
- 「某遥感团队的 Urban Forest 提取 pipeline」
- 「a16z 最新的 AI Agent 投资报告翻译」

**现状**：随手点收藏，到实验室已经忘了收藏了什么。微信收藏夹 500+ 条，从不回看。

**UC 场景**：把链接丢进 UC，AI 自动分类（AI技术/科研/投资）+ 生成一句话摘要。到实验室打开 Agent 问一句「今早收了什么」，三篇文章已排好队。

**10:00 — 组会/导师讨论**
导师转了一篇 arXiv 链接 + 一个会议 CFP（deadline 下周五）。

**现状**：arXiv 链接丢到 interested_papers.md，CFP 复制到微信文件传输助手。三天后发现 interested_papers.md 已经 200+ 行，CFP deadline 过了。

**UC 场景**：链接丢 UC → 自动识别「arXiv 论文」标签 + 「会议 CFP」标签 → CFP 自动提取 deadline → Agent 主动提醒「你收藏的 XX 会议还有 3 天截止」。

**14:00 — 写论文过程中查资料**
写到 Knowledge Representation 部分时，想起两周前看过一篇关于 knowledge graph + LLM 的好文章，但想不起标题和来源。

**现状**：花 20 分钟在微信收藏夹、浏览器历史、Notion 各个 page 里翻找，最后放弃。

**UC 场景**：直接问 Agent 「我收藏过关于 knowledge graph 和 LLM 结合的文章吗？」→ Agent 从知识库检索到 3 条相关收藏，附原始链接和当时生成的摘要。

**20:00 — 晚间浏览**
Twitter 上看到一个 Agent 框架的 thread（有人对比了 AutoGen vs LangGraph vs CrewAI），截图发给自己。又看到一篇 Web3 项目的 TVL 分析。

**现状**：截图存在相册，link 没保存。一周后写代码时想参考那个对比，找不到了。

**UC 场景**：复制链接丢 UC → 自动打标签「Agent框架」「对比分析」+「Web3/投资」→ 未来写实验系统时 Agent 能直接调取这个对比分析作为上下文。

### 核心痛点

1. **收藏即坟墓**：95% 的收藏再也不会被打开
2. **信息源分散**：微信、Twitter、邮件、arXiv、浏览器，五条管道互不相通
3. **检索靠记忆**：记得「看过」但不记得「在哪看」的，基本找不回来
4. **时间敏感信息被淹没**：deadline、CFP、申请通知等有时间窗口的信息，混在一堆无时效性的内容里
5. **知识无法复用**：收藏的信息无法被写作工具、Agent 系统、实验代码直接消费

### UC 价值

- **从「存」到「用」**：收藏不再是终点，而是知识流的入口
- **Agent 是大脑的外包**：不用记住「在哪看过」，问 Agent 就行
- **时效感知**：自动识别 deadline 类信息，主动提醒
- **写作助手**：论文写作时 Agent 能从收藏中检索相关线索，补充引用和论据

## Persona 2：陈维 — AI 独立开发者 / 创业公司 CTO

### 基本信息

| 维度 | 描述 |
|------|------|
| 年龄 | 30 |
| 身份 | AI SaaS 创业公司 CTO（5 人团队），同时自己做 side project |
| 城市 | 深圳 |
| 技术水平 | 极高。全栈 + ML，长期关注 AI infra 和 Agent 工具链 |
| 信息饮食 | 英文为主，Twitter/X 重度用户，RSS 订阅 30+ 博客，HN 每日刷 |
| 工具栈 | Chrome、Arc 浏览器、TweetDeck、GitHub、Linear、Notion、Raycast |

### 典型的一天

**08:00 — 晨间信息扫描**
刷 Hacker News + Twitter：
- 看到一个新的 MCP Server 实现（GitHub 2k stars）
- 有人发了 RAG vs GraphRAG 的深度对比文章
- OpenAI 发布了一个新的 function calling 最佳实践
- 某竞品拿到了 Series A 融资

**现状**：HN 用浏览器书签，推文截图发 Notion，GitHub repo 直接 star。一个月后 star 了 200+ repo，书签文件夹 10+ 层，Notion 里复制粘贴了一堆没整理的文本。真正需要参考时，还是得重新 Google。

**UC 场景**：所有链接统一丢 UC → 自动分为「开源项目/技术深度/竞品动态/AI产品」→ 一周后评估技术选型时，Agent 直接检索「最近收藏的 RAG 相关内容」，生成一张对比表。

**11:00 — 技术选型讨论**
团队在讨论 Agent 框架选型。陈维记得上个月看到过一篇 LangGraph vs AutoGen 的对比，还收藏了，但具体内容记不清了。

**现状**：打开 Notion 搜「LangGraph」—— 没找到。打开浏览器历史——太多记录。打开 GitHub stars——200+ 个 repo 不好翻。最后在 Twitter 搜索里翻了 10 分钟才找到那条 thread。

**UC 场景**：问 Agent「我收藏过 Agent 框架的对比分析吗？」→ 秒级返回那条 thread 的摘要 + 原始链接 + 当时提取的关键对比维度。

**15:00 — 写技术方案文档**
在写 RAG pipeline 的架构文档，需要引用几篇技术博客的结论。

**现状**：开了 20+ 个浏览器 tab，来回复制粘贴，但原文的关键论点已经忘了。还得重新读一遍。

**UC 场景**：Agent 已经有所有收藏的结构化摘要 → 「帮我整理收藏中关于 RAG pipeline 设计的关键结论」→ 直接生成一段结构化引用，附带来源链接。

**21:00 — 晚间投资/行业追踪**
关注了几家 AI infra 公司的动态，也在看 Web3 项目。

**现状**：竞品信息散落在 Notion、邮件、Twitter 各处。想做个竞品对比表，手动整理要 2-3 小时，一直拖着没做。

**UC 场景**：所有竞品相关信息已在 UC 中自动分类 → 「生成最近一个月我收藏的 AI infra 公司动态汇总」→ Agent 直接输出一份带时间线的竞品追踪报告。

### 核心痛点

1. **信息输入极度碎片化**：HN/Twitter/RSS/GitHub/邮件/播客，6+ 条管道
2. **技术选型时信息不可追溯**：记得「看过」但找不回原文，被迫重新搜索
3. **竞品追踪缺乏系统性**：散落的信息无法自动聚合为洞察
4. **side project 和公司项目信息混在一起**：缺少自动分上下文的能力
5. **收藏的知识无法直接注入工作流**：技术文档写作、架构决策时，收藏的信息是「死的」

### UC 价值

- **统一信息入口**：所有来源的内容走同一个管道，自动打标签分上下文
- **技术选型加速器**：Agent 帮你从收藏中检索和对比，不再靠记忆力
- **竞品情报自动化**：自动聚合 → 自动对比 → 自动生成报告
- **工作流注入**：收藏的知识可直接被 Agent 引用到技术文档、架构方案中

## Persona 3：苏雨 — B站/小红书 AI 领域内容创作者

### 基本信息

| 维度 | 描述 |
|------|------|
| 年龄 | 27 |
| 身份 | 全职 AI 领域内容创作者，B站 5 万粉 + 小红书 3 万粉 |
| 城市 | 上海 |
| 技术水平 | 中等偏高。非程序员出身，但会用 ChatGPT/Claude 辅助写作，懂基本的 prompt engineering |
| 信息饮食 | 中文为主，微信公众号 + 知乎 + Twitter 中文圈 + YouTube；日均浏览 30-50 条内容 |
| 工具栈 | 微信、小红书 APP、B站创作中心、飞书文档、Figma、剪映 |

### 典型的一天

**09:00 — 选题调研**
准备做一期「2026 年最值得关注的 5 个 AI Agent 框架」视频。需要搜集素材。

**现状**：打开 10+ 个浏览器 tab（各框架官网、GitHub、测评文章），微信公众号收藏了 8 篇相关文章，飞书文档里新建了一个「选题素材」页面手动复制链接。但素材来源分散，每次做新选题都要从零开始搜集。

**UC 场景**：日常浏览时看到好的框架分析就丢 UC → 自动分类到「AI Agent 框架」→ 做选题时问 Agent「我收藏过哪些 Agent 框架的分析？」→ 直接获得素材清单 + 每篇的关键观点摘要。

**13:00 — 内容创作**
在写视频脚本。需要引用一些具体数据和观点。

**现状**：之前收藏的文章和数据散落在微信、浏览器书签、飞书文档各处。找数据时经常找不到确切来源，只能凭记忆写一个模糊的说法，或者花 30 分钟重新搜索。

**UC 场景**：Agent 已有所有收藏的结构化摘要 → 「帮我整理收藏中关于 XX 框架的用户增长数据和性能对比」→ 秒级返回带来源的数据摘要。

**16:00 — 社交媒体运营**
小红书需要日更 1-2 条。要找当天的热点话题。

**现状**：刷 Twitter 和公众号，看到有趣的就截图。但要发帖时，经常发现截图找不到或者忘了上下文。

**UC 场景**：日常浏览随手收藏 → UC 自动生成摘要和标签 → 做 daily post 时，Agent 建议「今天你收藏的热点话题有 X、Y、Z，最热门的是 X，要帮你生成一条小红书笔记草稿吗？」

**20:00 — 长期选题规划**
想做一个「AI 产品月报」系列，需要系统追踪行业动态。

**现状**：靠脑子记 + 临时搜索。没有系统性的信息归档。上个月的行业动态已经记不清了。

**UC 场景**：UC 自动按月归档收藏 → Agent 月底自动生成「本月 AI 产品动态汇总」→ 直接作为月报素材。

### 核心痛点

1. **选题素材散落各处**：每次做新内容都要从零搜集，无法复用历史收藏
2. **引用和数据无法溯源**：写内容时找不到确切来源，影响内容可信度
3. **热点追踪靠人力**：没有自动化的信息聚合，效率低
4. **知识无法沉淀为选题库**：收藏的内容没有结构化，无法直接转化为选题灵感
5. **跨平台内容管理困难**：微信、B站、小红书各有各的信息流，难以统一管理

### UC 价值

- **选题素材自动积累**：日常浏览即选题准备，不需要额外整理
- **数据溯源**：每个观点都能追溯到原始来源，内容更可信
- **热点自动追踪**：Agent 主动推送收藏中的热点趋势
- **内容生产加速**：Agent 从收藏中生成内容草稿，创作者只需要润色
- **选题库沉淀**：收藏按主题自动聚类，形成可检索的选题灵感库

## 三个 Persona 的交叉洞察

| 维度 | Persona 1（博士生） | Persona 2（开发者/创始人） | Persona 3（创作者） |
|------|---------------------|--------------------------|---------------------|
| **收藏频率** | 5-10 条/天 | 15-20 条/天 | 10-15 条/天 |
| **主要信息源** | 微信、arXiv、Twitter | HN、Twitter、RSS、GitHub | 微信、小红书、知乎、YouTube |
| **核心焦虑** | 信息浪费 + deadline 错过 | 技术选型无据 + 竞品盲区 | 选题枯竭 + 素材不可溯源 |
| **回看率** | <5% | <8% | <10%（但需求更高） |
| **Agent 使用意愿** | 高（已经在用） | 极高（技术背景） | 中等（需要极简交互） |
| **付费意愿** | 低（学生预算） | 中高（公司报销或 side project 预算） | 中（创作者工具预算 $10-20/月） |
| **关键使用场景** | 论文写作时检索收藏 | 技术选型和竞品追踪 | 选题调研和内容生产 |

### 共同的底层需求

1. **收藏不应是终点**——收藏只是信息流的入口，后续的加工和复用才是价值
2. **记忆外包给 AI**——不需要记住「在哪看过」，AI 帮你检索
3. **信息需要自动结构化**——手动分类不可持续，AI 分类必须足够好
4. **知识需要可注入**——收藏的信息要能直接为下游任务（写作、决策、内容生产）服务
5. **低摩擦是前提**——任何需要超过 2 步操作的收藏流程都会被放弃
